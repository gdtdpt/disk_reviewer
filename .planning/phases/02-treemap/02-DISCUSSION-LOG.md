## 布局算法

| 选项 | 描述 | 选中 |
|------|------|------|
| 自研实现 | 从零实现 Squarified 算法，完全掌控布局和性能，无额外依赖 | ✓ |
| 使用现有 crate | 查找 treemap crate，减少开发时间但可能不匹配 egui 需求 | |
| 简化版切片布局 | slice-and-dice 算法，实现简单但矩形长宽比差 | |

**User's choice:** 自研实现（推荐）
**Notes:** 用户认可 ~200 行的实现量，学习价值高。

| 选项 | 描述 | 选中 |
|------|------|------|
| 独立 TreemapNode | 创建独立结构 rect(x,y,w,h)+label+color+depth+Entry引用，布局与数据解耦 | ✓ |
| 扩展现有 Entry | 在 Entry/DirNode 上添加 rect 字段，污染扫描层数据模型 | |
| HashMap 索引 | HashMap<PathBuf, Rect>，查找灵活但内存开销大 | |

**User's choice:** 独立 TreemapNode（推荐）

| 选项 | 描述 | 选中 |
|------|------|------|
| 消费 DirNode 树 | 布局算法接收 DirNode 递归树，每次下钻重新运行，只布局当前 children | ✓ |
| 预计算全量布局 | 扫描完成后一次性计算所有层级，内存大但切换无延迟 | |
| 按需增量布局 | 只布局可见区域，类似虚拟滚动，实现复杂 | |

**User's choice:** 消费 DirNode 树（推荐）

## 颜色 + 标签

| 选项 | 描述 | 选中 |
|------|------|------|
| 按文件类型 | 不同扩展名/类型分配不同色系（蓝=文档，绿=媒体，红=系统） | ✓ |
| 按目录深度 | 同层级同颜色，深度越深颜色越浅 | |
| 按大小渐变 | 大正方形深色，小正方形浅色 | |
| 按目录分组 | 同父目录子项用相近色系 | |

**User's choice:** 按文件类型（推荐）
**Notes:** 用户特别询问了目录颜色的处理方式。讨论后确定：目录用主导类型色（统计目录下各类型占比，用占比最高的类型颜色代表整个目录）。图例放在详情面板内。

| 选项 | 描述 | 选中 |
|------|------|------|
| 最小面积阈值 | 矩形面积小于阈值时不显示标签，悬停 tooltip 显示完整信息 | ✓ |
| 截断 + 省略号 | 无论多小都显示标签，超出截断为 '...' | |
| 仅显示大小 | 小矩形只显示大小数字 | |

**User's choice:** 最小面积阈值（推荐）
