# Phase 2 Plan 06: 侧边栏文件列表

## Goal
在右侧详情面板中增加当前目录下的文件/目录列表，为无法在 treemap 中点击的小文件提供替代交互入口。

## Background
Treemap 中小色块（面积 < 200 px²）不显示标签，用户难以确认其存在和位置。文件列表提供线性视图，与色块共享选中/下钻状态。

## Tasks

### Task 1: 文件列表组件 `src/ui/file_list.rs`
**Goal**: 创建可滚动的文件列表，显示颜色标识、名称、占比、大小

**Acceptance Criteria**:
- [ ] `file_list_ui(ui, nodes, selected_index, on_select, on_drill)` 函数
- [ ] 列表项水平布局：12x12 颜色色块 + 名称（左对齐）+ 占比（右对齐）+ 大小（右对齐）
- [ ] 使用 `ScrollArea::vertical()` 支持滚动
- [ ] 选中项高亮背景（与 treemap 金色高亮一致）
- [ ] 单击回调 `on_select(index)`
- [ ] 双击回调 `on_drill(index)`
- [ ] 目录项显示文件夹图标前缀，文件显示颜色色块前缀

**Implementation Notes**:
- 接收 `&[TreemapNode]` 作为数据源（与 treemap 共享同一份数据）
- 通过回调与父组件通信，不直接修改 app 状态
- 字体使用 proportional 12pt（与全局 style 一致）

### Task 2: 集成到详情面板
**Goal**: 将文件列表嵌入右侧 info_panel，位于详情信息和颜色图例之间

**Acceptance Criteria**:
- [ ] `info_panel.rs` 中调用 `file_list_ui()`
- [ ] 列表与 treemap 共享 `selected_index`
- [ ] 列表选中 → treemap 高亮同步
- [ ] treemap 选中 → 列表高亮同步
- [ ] 双击列表项目录 → 下钻（与 treemap 双击行为一致）
- [ ] 单击列表项 → 选中（与 treemap 单击行为一致）

**Implementation Notes**:
- 修改 `info_panel_ui` 签名，增加 `nodes` 和 `selected_index` 参数
- 在 `app.rs` 中传递对应数据

### Task 3: app.rs 集成
**Goal**: 连接 treemap 和文件列表的选中/下钻状态

**Acceptance Criteria**:
- [ ] `info_panel_ui` 调用时传入 `&self.treemap_nodes` 和 `self.selected_index`
- [ ] 列表单击事件 → 更新 `selected_index`
- [ ] 列表双击事件 → 调用 `drill_down()`
- [ ] 下钻后列表自动滚动到顶部

**Implementation Notes**:
- 复用现有的 `drill_down()` 和 `selected_index` 逻辑
- 不需要新增状态，仅增加回调传递
