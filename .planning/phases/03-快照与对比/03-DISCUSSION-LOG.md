## 快照存储格式

| Option | Description | Selected |
|--------|-------------|----------|
| 整树 JSON | 整个 DirNode 序列化为 JSON 存单条记录，实现最简单 | |
| 关系表 | 每个目录/文件独立行，查询灵活但实现复杂 | |
| 混合模式 | 元信息存关系表，目录树存 JSON 列 | |
| **路径索引 + JSON 子树** | 每个目录节点单独存一条记录，key 为完整路径，value 为该节点子树 JSON | ✓ |

**User's choice:** 路径索引 + JSON 子树，快照替换时整体清理
**Notes:** 用户关注效率——只对比当前显示路径的差异，不需要加载整棵树。每个快照有唯一 ID，保存时先删旧记录再写入，避免垃圾数据。快照默认名称带创建时间（如 `快照 2026-05-06 14:30`）。

## 差异检测策略

### 匹配方式

| Option | Description | Selected |
|--------|-------------|----------|
| **按名称匹配** | 同一层级按条目名称匹配，简单直接 | ✓ |
| 按相对路径匹配 | 从快照根到当前路径的相对路径匹配 | |
| 按完整路径匹配 | 完整绝对路径匹配，最严格 | |

**User's choice:** 按名称匹配

### 变化类型

| Option | Description | Selected |
|--------|-------------|----------|
| **四种** | 新增、删除、增长、缩小 | ✓ |
| 三种 | 新增、删除、变化（增长缩小合并） | |
| 两种 | 新增、删除 | |

**User's choice:** 四种（新增、删除、增长、缩小）

## 差异高亮方式

| Option | Description | Selected |
|--------|-------------|----------|
| 叠加半透明色 | 原有矩形上叠加半透明色层 | |
| 边框高亮 | 给有差异的矩形加粗边框 | |
| **独立对比窗口** | 新窗口左右并排显示当前结果和快照，快照侧标识状态 | ✓ |

**User's choice:** 独立对比窗口，左右并排。快照侧色块叠加半透明色（新增=绿、删除=红、增长=橙、缩小=蓝），角落加小图标（+、-、↑、↓），鼠标悬停 tooltip 显示变更详情（名称、旧大小、新大小、变化量）。
